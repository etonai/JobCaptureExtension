## LinkedIn Job Capture Extension (Microsoft Edge)

### Project Goal

Create a Microsoft Edge (Chromium) browser extension that captures information from the currently displayed LinkedIn Job page and stores it in a structured format for a local job tracking system.

This extension is **not** intended to automate job applications.

Its purpose is to preserve job postings and create a searchable history of the user's job hunt.

The extension should rely on deterministic parsing of the page contents. **No AI or LLM parsing should be used.**

---

# Overall Philosophy

The extension should do one thing well:

> Capture the current LinkedIn job posting.

Everything else (analysis, statistics, resume management, interview tracking, etc.) will be handled by another application later.

---

# Initial Scope (MVP)

When the user clicks the extension button while viewing a LinkedIn Job page, the extension should:

1. Verify that the current page is a LinkedIn Job page.
2. Expand the job description if necessary.
3. Extract structured information.
4. Extract the complete job description.
5. Present the extracted information for review.
6. Allow the user to save/export the captured record.

---

Capture the text exactly as displayed whenever practical.

Fields:

* Posted Text
* Applicant Count Text

* Hiring Status Text

Examples:

## Compensation

Capture:

Salary Text

Examples:

```
$127K/yr - $211K/yr
```

Do **not** parse into numeric values in the MVP.

Store the visible text exactly.

---

## Apply Type

Determine which type of application this posting uses.

Possible values:

* Easy Apply
* External Apply
* Unknown

---

## Job Description

Capture the complete job description.

Before extraction:

* Detect whether the description is collapsed.
* If a "...more" or equivalent expansion control exists, expand it.
* Wait for the page to update.
* Capture the full text.

Capture these if present.

Entire section as text.

# User Interface

For the MVP, a simple popup is sufficient.

Suggested flow:

```
Capture Job

↓

Read page

↓

Extract fields

↓

Display extracted values

↓

User may edit fields

↓

Save
```

Editing should be supported before saving, as extraction may occasionally be imperfect.

---

# Parsing Strategy

The implementation should prioritize:

1. Stable DOM structure.
2. Semantic elements.
3. Visible labels.
4. Text patterns.

Avoid depending heavily on LinkedIn CSS class names because they frequently change.

The parser should tolerate minor page layout changes.

---

# Error Handling

If a field cannot be extracted:

* Leave it blank.
* Do not abort the capture.
* Continue capturing all remaining fields.

Capturing an incomplete record is preferable to losing the job posting.

# Development Notes

The extension should be written as a standard Chromium (Manifest V3) extension compatible with Microsoft Edge.

The implementation should emphasize:

* Maintainability
* Readability
* Robustness against LinkedIn UI changes
* Deterministic extraction
* Minimal external dependencies

The capture logic should be isolated from the user interface so that it can later be reused by other tools or applications.
